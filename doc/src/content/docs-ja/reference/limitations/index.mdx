`design-token-lint` は正規表現ベースの静的解析を使用して、ソースファイルからクラス名を抽出します。このアプローチは高速でビルドステップが不要ですが、実行時にコードを評価することはできません。そのため、一部の動的パターンは抽出されず、リントされません。

これらはバグではなく、静的解析に固有の制限です。

## サポート済み: 複数行の className

複数行の `className` 値はサポートされています:

```tsx
<div
  className="
    p-4
    bg-gray-500
  "
>
```

複数行にまたがるクラスは正しく抽出されます。

## 制限事項: 条件式

`className` 内の三項演算子は抽出されません:

```tsx
// リントされない — 三項演算子内のクラスは抽出されない
<div className={isActive ? "p-4" : "m-8"}>
```

エクストラクターは `className` や `class` に直接割り当てられた文字列リテラルを探します。三項演算子の構文にはマッチしません。

**回避策**: クラスがフラグされる場合は無視コメントを使用してください:

```tsx
{/* design-token-lint-ignore */}
<div className={isActive ? "p-4" : "m-8"}>
```

## 制限事項: テンプレート補間

動的な式を含むテンプレートリテラルは抽出されません:

```tsx
// リントされない — 動的なテンプレートリテラルは静的解析できない
<div className={`p-${size} bg-${color}-500`}>
```

すべてのケースで静的な部分と動的な部分を確実に分離することはできません。

**回避策**: 無視コメントを使用するか、静的なクラス名と条件ロジックを使用する構造に変更してください。

## 制限事項: エスケープされた引用符

エスケープされた引用符を含むクラス属性は正しく抽出されない場合があります:

```tsx
// 正しく抽出されない場合がある
<div className="p-4 \"m-8\"">
```

正規表現パーサーは文字列リテラル内のエスケープされた引用符シーケンスを処理しません。

**回避策**: クラス属性内でエスケープされた引用符を避けてください。代わりに JSX の式構文を使用してください:

```tsx
<div className={'p-4 m-8'}>
```

## 制限事項: class:list のオブジェクトキー

Astro の `class:list` ディレクティブはキーがクラス名のオブジェクト構文をサポートしています。これらは抽出されません:

```astro
// リントされない — オブジェクトキー構文は抽出されない
<div class:list={[{ "p-4": true, "m-8": isActive }]}>
```

`class:list` では配列の文字列リテラルのみがサポートされています:

```astro
// これはサポートされている
<div class:list={["p-4", "m-8"]}>
```

**回避策**: 無視コメントを使用するか、配列文字列形式を使用する構造に変更してください。

## まとめ

| パターン | サポート |
| --- | --- |
| 複数行の `className` | あり |
| 静的な文字列リテラル | あり |
| テンプレートリテラル(静的のみ) | あり |
| 三項演算子 | なし |
| 補間付きテンプレートリテラル | なし |
| クラス文字列内のエスケープされた引用符 | なし |
| `class:list` のオブジェクトキー | なし |
